## 1) خلاصة الاتجاه الذي استقرّ عليه المشروع

الدردشة بدأت من نقطة “توليد فيديو وصور” وتفرعت إلى حلول كثيرة. في النهاية أنت وصلت لحل **أبسط وأقوى**:

### ✅ التحول الأساسي

بدل ما تبني **محركات إنتاج** (Video/Image APIs + صلاحيات + مشاكل سياسات المنصات)
أنت تبني **بيئة تشغيل** (Workspace) داخلها المستخدم يشتغل بحسابه وأدواته، و”وياك” يصير **وكيل تنفيذي** يحول كل طلب إلى مهام ويُنجزها داخل البيئة.

---

## 2) المفهومين الأساسيين: “كمبيوتر وياك” و “متصفح وياك”

### A) كمبيوتر وياك (Weyaak Computer)

هو **واجهة سطح مكتب كاملة داخل المتصفح** (Cloud Desktop / Virtual Desktop) مثل اللي بالصورة اللي عرضتها.

**فائدته:**

* المستخدم يحصل على “كمبيوتر جاهز” يفتح من أي جهاز.
* يقدر يثبت/يضيف تطبيقات ويب كـ **Apps** داخل Edge (Install as App / PWA).
* يشتغل على WhatsApp Web، Facebook، Google Drive، Canva… إلخ.
* أهم نقطة: **ما تحتاج API ولا صلاحيات دخول** لأن المستخدم هو من يسجّل دخول بنفسه داخل الجلسة.

**ملاحظة:** هنا كلمة “تنزيل تطبيق” غالبًا تعني:

* تثبيت “تطبيق ويب” داخل Edge (Install site as app)
  وليس تنزيل تطبيق iOS/Android من المتجر داخل سطح مكتب.

---

### B) متصفح وياك (Weyaak Browser)

هو **متصفح داخل النظام** يكون أساسه Edge، لكن مزود بـ:

* زر ثابت “اكمل وياك”
* لوحة مهام
* وكلاء متخصصين ينفذون خطوات داخل المتصفح

**الفرق المهم:**

* في PWA وحدها لا تقدر تبني متصفح حقيقي يتحكم بكل المواقع داخل نفس الإطار (قيود iframe/CSP).
* عشان “متصفح وياك” يكون فعلاً متصفح قابل للأتمتة + زر ملازم فوق كل شيء، الأفضل يكون ضمن:

  1. **كمبيوتر وياك** (الأقوى)
     أو
  2. تطبيق ستور (WebView/WebView2) — لكن أنت لاحقًا ركزت أن “كمبيوتر وياك” هو الحل.

## 3) لماذا هذا حل “ينهي” عقد الفيديو والصور؟

أنت قلت: *“عقدة الفيديو والصور انحلت”* — وهذا صحيح لأنك نقلت الإنتاج إلى أدوات المستخدم:

### نموذج التشغيل الجديد:

* **وياك** يجهز: سكربت + أفكار + نصوص + خطة تحرير + قوالب
* المستخدم داخل **كمبيوتر وياك** يفتح أدواته (Canva/CapCut/غيرها)
* يتم “الإنتاج/التصدير” هناك

✅ النتيجة:
لا تكلفة GPU عندك، ولا APIs فيديو ثقيلة، ولا تعقيد ربط.

---

## 4) الفكرة الذهبية: “بدون API وبدون صلاحيات دخول”

أنت حسمتها:

> المستخدم يفتح واتساب/فيسبوك/جوجل داخل متصفح وياك، وهو يسجل دخول… وانتهينا.

**هذا يقلل:**

* مخاطر الخصوصية
* مشاكل موافقات Meta/WhatsApp API
* الصداع التشغيلي

**لكن شرط أساسي للنجاح:**

* الوكيل لا “يتجاوز” تسجيل الدخول أو Captcha أو 2FA
  إذا ظهر، المستخدم يكملها، ثم الوكيل يكمل.

---

## 5) أهم مبدأ تشغيلي: “كل رسالة أو طلب = مهمة”

هذا هو قلب مشروع “وياك التنفيذي”.

### بدل:

سؤال → رد

تصير:
طلب → **Task** → Workflow → نتيجة + توثيق + احتساب رصيد

**أثره عليك:**

* تقدر تحسب الاستهلاك بالمهام (Coins/رصيد)
* تقدر تدير التشغيل والمتابعة
* تقدر تبني لوحة مهام للمستخدم والمزود

---

## 6) بنية الوكلاء التي اتفقنا عليها

أنت قررت:

> كل مهمة لها وكيل مخصص، والوكيل يقدر ينشئ سير عمل مخصص.

### الهيكل الصحيح:

* **وياك الرئيسي (Supervisor / Orchestrator)**
  يفهم الطلب، يختار الوكيل، ينشئ المهمة، يطلب موافقة المستخدم، ويتابع.

* **وكلاء مهام (Task Agents)**
  كل وكيل متخصص بمجال:

  * وكيل الحسابات (Account Agent)
  * وكيل SEO/الظهور (SEO & Visibility Agent)
  * وكيل التسويق الرقمي (Marketing Agent)
  * وكيل التعليقات والرسائل (Engagement Agent)
  * وكيل واتساب ويب (WhatsApp Web Agent)
  * وكيل المحتوى (Content Agent)
  * وكيل الفيديو (Video Plan/Packaging Agent)

## 7) “وكيل الحسابات” الذي تريد بناءه (أهم وكيل الآن)

أنت قلت إن باقة 299 تعطي:

* رابط ملف شخصي متوافق مع بحث الذكاء الاصطناعي
* * رابط عن النشاط (خدمة/صفحة)

وهذا بالضبط عمل **وكيل الحسابات**:

### وظيفة وكيل الحسابات من أول دخول لأول مرة:

1. يجمع بيانات النشاط بسرعة (اسم، مدينة، نشاط، هاتف، واتساب، روابط…)
2. ينشئ “Business Profile” موحد داخل بيت الريف
3. ينتج صفحتين:

   * **صفحة الملف التجاري** (Profile Page)
   * **صفحة الخدمة** (Service Page)
4. يجهّز “تهيئة AI/SEO” لهذه الصفحات (بدون تعقيد للمستخدم)
5. يسلّم المزود روابط جاهزة للنشر والمشاركة

**وهذا يفتح الطريق مباشرة للوكيل التالي:**
✅ وكيل التسويق الرقمي

## 8) وكيل SEO والقياس الذي صار هدفك الرئيسي الآن

أنت وضحت الهدف الحالي:

> قياس نشاط العميل على الذكاء الاصطناعي وعلى جوجل + شكل تقارير + حلول واضحة بمدة زمنية.

وهذا يتحول إلى “منتج تقارير” قوي جدًا.

### ما الذي نقيسه؟

* كيف يظهر النشاط في Google Search (ومنها AI Overviews إن ظهرت)
* Google Maps presence (ملف النشاط، الصور، التقييمات)
* جودة الموقع تقنيًا (أداء/سرعة/SEO أساسي)
* “AI presence” (هل يظهر النشاط في إجابات الذكاء الاصطناعي لما نسأل أسئلة محلية؟)

### شكل التقرير المطلوب (احترافي مثل الصور)

ليس كلام عام. بل:

* بطاقات نتائج
* روابط مباشرة
* مقارنة
* توصيات
* خطة زمنية

## 9) التقنيات المقترحة حسب كل جزء

هنا الجزء اللي طلبته: **ذكر التقنيات + فكرة متصفح وياك** بشكل واضح ✅

### A) منصة الأتمتة والمهام

* **n8n**: Orchestration / Workflows / Queue / Logging
* **Supabase**: Auth + RBAC + DB (Tasks/Projects/Profiles) + تخزين Metadata
* **Coins/Entitlements**: داخل Supabase (جداول + سياسات) وربطها بتنفيذ n8n

### B) كمبيوتر وياك (Cloud Desktop)

حسب بيئتك:

* **noVNC** أو **Apache Guacamole** لعرض سطح المكتب في المتصفح
  (يعطي تجربة “كمبيوتر داخل المتصفح” مثل الصورة)
* نظام تشغيل:

  * Linux Desktop (XFCE/Ubuntu) كخيار اقتصادي
  * أو Windows VM لو احتجت تطبيقات ويندوز فعلًا

### C) متصفح وياك (داخل كمبيوتر وياك)

* **Microsoft Edge** كمتصفح افتراضي داخل البيئة
* استخدام ميزة **Install site as app (PWA)** لتثبيت WhatsApp Web / Canva / Facebook كـ “Apps”

### D) تنفيذ الوكلاء داخل المتصفح (Agent Execution)

بدون APIs خارجية، التنفيذ يكون عبر أتمتة المتصفح داخل نفس البيئة:

* **Playwright** (ممتاز وقوي)
* أو **Selenium**
* أو **Puppeteer**

**فكرة التنفيذ:**

* الوكيل يفتح صفحة/تطبيق ويب
* يراقب أحداث (تعليقات/رسائل) بطريقة آمنة (Polling خفيف أو UI checks)
* يقترح رد
* ينفذ بعد موافقة المستخدم

### E) قياس SEO/Google

* **Google PageSpeed Insights** (للتحليل التقني للأداء)
* فحص عناصر SEO الأساسية (Title/Meta/H1/Schema)
* بناء صفحات “Profile/Service” مجهزة بـ:

  * **Schema.org** (LocalBusiness/Service/Organization)
  * OpenGraph
  * محتوى منظم

### F) ربط “كلاود المستخدم”

أنت اخترت: “بيانات المستخدم عنده”
والحل الأقوى الآن بدون APIs:

* المستخدم يسجل دخول إلى **Google Drive / OneDrive / iCloud** داخل Edge
* ويحفظ ملفاته هناك

(لاحقًا إن احتجت “تصدير تلقائي”، وقتها يكون OAuth/API، لكن ليس ضروري كبداية.)

## 10) كيف تتحكم في الاستهلاك والرصيد في هذا النظام؟

الآن صار سهل لأن:

* كل شيء “Task-based”

### نموذج واضح:

* Task: “رد على تعليق” → تكلفة ثابتة
* Task: “تقرير ظهور” → تكلفة ثابتة
* Task: “إعداد صفحة خدمة” → تكلفة ثابتة
* Task: “خطة 30 يوم” → تكلفة ثابتة

ويتم خصم الرصيد:

* عند إنشاء المهمة
* أو عند خطوة “Generate” داخل المهمة
  حسب ما تحدده

## 12) ترتيب البناء المقترح (أبسط طريق للواقع)

2. وكيل الحسابات (Onboarding + صفحات Profile/Service)

### المرحلة 3 (كمبيوتر وياك بالكامل)

6. توحيد تجربة “كمبيوتر وياك” وربط المتصفح Edge

## 13) ما الذي نحتاجه منك الآن لتثبيت التنفيذ بسرعة؟

بدون أسئلة كثيرة، فقط 3 قرارات:

1. **كمبيوتر وياك**: Linux Desktop ولا Windows؟
   (أنا أنصح Linux كبداية: أرخص وأخف)
2. أول منتج نبيعه:

   * تقرير الظهور + صفحة Profile/Service (باقة 299)
3. أول وكيلين:

   * Account Agent
   * SEO/Visibility Agent

* **هيكلة بيانات “Business Profile”**

* **قالب تقرير جاهز** (مثل الصور: بطاقات + روابط + خطة زمنية)
